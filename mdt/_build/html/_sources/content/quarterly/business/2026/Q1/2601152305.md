# LLM 原生场景仍未出现

自 2022 年 OpenAI 发布 ChatGPT 以来，全球市场已经将数十万亿美元级别的资金绑定在了以 LLM 为基础技术的产业链上，整条产业链从上游到下游主要可以抽象为：

> （上游基建）数据中心 -> （中游平台）云服务 -> （下游应用）SaaS

其中属于上游基建的数据中心虹吸了最主要的资金，但是中游平台和下游应用能否通过主营业务盈利回收上游基建的投入呢？本轮周期恐怕非常困难，因为至今依然没有属于 LLM 的原生场景出现。

## 技术原生场景

什么是技术原生场景？

技术创新自带价值链的应用场景，而非附加技术的改造，即如果没有该技术，这个场景在商业与体验上不可成立。
用户对使用该技术应用场景有强烈意图，而非被动接受。

那么目前 LLM 主要应用的市场定位是什么？

1. aggregated searching engine
2. chatbot
3. coding pilot
4. content generator

这 4 大类场景早在 ChatGPT 发布前就已经有相对成熟的应用，LLM 作为基础技术的 GPT 只是增强了其性能，且这 4 大类场景目前依然只有 coding copilot 产品基本实现了盈利，其余依然处于亏损状态。这就说明截至目前仍没有诞生 LLM 原生场景，导致用户不愿意为其进行高溢价付费。

下面列举一些技术原生场景进行对比分析：

1. 互联网搜索：Google，改变用户获取信息的方式。
2. 社交网络：Facebook，构建用户个人身份与社交关系。
3. 推荐算法：Tiktok，将用户主动获取信息改为向用户投喂信息。

以上这些技术的商业化属于开创了新的应用场景，而且无一例外几乎都是通过扩大用户规模的同时收取广告费用来盈利。

## 原生场景已经在酝酿只是未爆发

> We tend to overestimate the effect of a technology in the short run and underestimate the effect in the long run. —— Roy Amara

[![](./img/Hype-Cycle-General.png)](https://www.gartner.com/en/research/methodologies/gartner-hype-cycle)

LLM 的技术原生场景一定也符合 LLM 的显著特点。

1. **高度容错**：由于 LLM 生成的内容是逻辑自我闭环但不一定符合实际逻辑，所以如果用户对内容要求是低容错，那么使用 LLM 将不能有效发挥其最大效果。
2. **持续交互**：语言模型的最大用途就是持续和用户进行交流互动。
3. **形式化复杂**：对于形式化简洁清晰的任务，状态机比 LLM 的效果要相对更好。
4. **模糊价值**：LLM 与用户交互过程的价值远高于可明确量化结果的价值，但是过程的价值是模糊的。

### 明确排除场景

1. **机器人**：由于机器人所需要使用的训练数据和控制算法必须符合现实世界的物理定律，使用 LLM 生成的逻辑自我闭环内容很可能违反物理定律，因此不适用。
2. **XR/VR/AR/MR**：XR 等解决的是模拟现实世界的空间和感知，与上述机器人相同，也有相当程度的物理定律约束，因此 LLM 只能增益其效果而不是决定性的。

### 展望

1. **决策仿真与个性化教育**：决策和学习的过程相比结果更加重要，且通常涉及可能不存在的逻辑闭环，需要通过 LLM 辅助修正。
2. **游戏**：由于用户可以接受游戏中新定义的逻辑且接受容错，所以高度匹配 LLM 的技术特点。
3. **情感陪伴**：用户对于虚拟的情感交互，更重视 LLM 生成的逻辑自我闭环的内容和过程，而并非确定性的结果。
